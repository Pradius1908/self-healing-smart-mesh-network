# JSON Communication Protocol

Every packet in the network (both wireless mesh and serial connection) must conform to the following JSON schema.

## Schema Definition

```json
{
  "id": "A_17203",
  "type": "HEARTBEAT",
  "source": "A",
  "destination": "ALL",
  "nextHop": "ALL",
  "priority": "ROUTINE",
  "timestamp": 17203,
  "payload": {}
}
```

### Protocol Fields

| Field Name | Type | Description |
| :--- | :--- | :--- |
| `id` | String | A unique packet identifier (e.g., `<source>_<timestamp>`). |
| `type` | String | The packet type. Supported types listed below. |
| `source` | String (Char) | The originating node ID ('A'-'E'). |
| `destination` | String (Char) | The final target node ID ('A'-'E'), or 'ALL' for broadcasts. |
| `nextHop` | String (Char) | The intermediate recipient ID ('A'-'E'), or 'ALL' for broadcasts. |
| `priority` | String | Priority level. Options: `ROUTINE`, `EMERGENCY`, `SYSTEM`. |
| `timestamp` | Unsigned Long | Timestamp in milliseconds since node boot (or epoch if synced). |
| `payload` | Object | Nested object containing type-specific data. |

---

## Packet Types

### 1. `HEARTBEAT`
Broadcasted every 5 seconds by every node. Used for discovery, address mapping, and link health tracking.
* **Payload Fields**:
  * `battery`: Integer (0-100 placeholder score).
  * `rssi`: Integer (Link signal strength in dBm, if available).
  * `chipId`: Unsigned Long (painlessMesh 32-bit hardware node ID).

### 2. `PING`
Used to measure Round Trip Time (RTT).
* **Payload Fields**:
  * `sentTime`: Unsigned Long (timestamp of original transmission).

### 3. `PONG`
Echo reply to a `PING` packet. Must carry the exact timestamp of the incoming `PING`.
* **Payload Fields**:
  * `sentTime`: Unsigned Long (copied from the corresponding `PING`).

### 4. `ROUTINE`
Standard telemetry and message logs (e.g., clinical logistics reports, local shelter stats).
* **Payload Fields**:
  * `text`: String (message body).

### 5. `EMERGENCY`
High-priority alert packets (e.g., patient triage, critical failure reports).
* **Payload Fields**:
  * `text`: String (emergency description).

### 6. `SET_ROUTE`
Command packet injected by the backend. Instructs a specific node to update its next-hop routing target.
* **Payload Fields**:
  * `dest`: String (Char) - The target destination node.
  * `next`: String (Char) - The assigned next-hop node letter.

### 7. `SET_MODE`
Command packet injected by the backend to transition a node or network state.
* **Payload Fields**:
  * `mode`: String (`NORMAL` or `EMERGENCY`).

### 8. `FORWARD`
An encapsulated packet that is in transit along a multi-hop path.
* **Payload Fields**:
  * Contains the original nested routine or emergency data.

### 9. `SYSTEM`
System status, network events, and general administrative alerts.

### 10. `NODE_DOWN` & `NODE_UP`
Generated by the backend and sent to the dashboard when node failures/recovery are detected.
